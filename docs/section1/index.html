Prerequisites

  • Bluemix and DevOps Services accounts
  • Eclipse mars for Java EE Developers
  • Eclipse plugins:
    • was liberty
    • bluemix
  • Chrome browser
  • Chrome plugin:
    • postman
  • Java JDK 1.7

Postman Chrome Plugin Installation

1.      Open Chrome browser, and browse to https://www.getpostman.com. Click Get in now – it’s free!
Note: Chrome browser is specifically needed here, as we are going to install Postman as Chrome plugin

 

2.      A window pop out. Click ADD TO CHROME

 

3.      A security warning window pop out. Select Add app

 

4.      Click LAUNCH APP button

 

5.      You should see the Welcome to Postman 3.0 page

 

 

 

WAS Liberty Eclipse Plugin Installation

1. Open the browser, and browse to https://developer.ibm.com/wasdev/downloads/liberty-profile-using-eclipse/. Open the Eclipse. Then, drag the  towards the Eclipse workspace.

 

2. We should see now the Eclipse Marketplace window. We are going to install all the features, so leave all the default values and click Confirm.

 

3. Select I accept the terms of the license agreements and click Finish

 

4. Download and installation process starts. Depending on the network bandwidth, this process might take several minutes to complete.

 

5. After installation complete, you will get prompted to restart the Eclipse. Select Yes.


 

 

WAS Liberty Server Configuration

1. At the Eclipse workspace, open the Servers tab, right click on any white space, and select New > Server

 

2. Expand IBM, then select WebSphere Application Server Liberty Profile. Leave all other fields default values, and click Next

 

3. Choose Install from an archive or a repository. For JRE, select use default JRE (currently ‘jdk1.7.0_79). Click Next

 

4. Make sure the destination path is point to C:\Users\IBM_ADMIN\WAS Liberty. Select Download and install a new runtime environment from ibm.com,

and choose WAS Liberty V8.5.5.7 with Java EE 7 Full Platform. Click Next

 

5. Just click Next to continue.

 

6. Select I accept the terms of all the license agreements, and click Next

 

7. Leave to all default values, and click Finish

 

8. After installation, you should see the successful message below. Click OK.

 

9. You should now see WebSphere Liberty Profile appearing stopped in the Servers tab. Select the WebSphere Application Server Liberty Profile at localhost [defaultServer], and click  to start the server

 

10. When get prompted on the security settings, verify and enter the following values:

Keystore password: passw0rd

Add basic user registry: checked

User name: demo

Password: demo

 

11. When you see the log message The server defaultServer is ready to run a smarter planet, this means the WebSphere Liberty Server has been successfully started.

 

 

Bluemix Eclipse Plugin Installation

1. Open the browser, and browse to https://marketplace.eclipse.org/content/ibm-eclipse-tools-bluemix. Open Eclipse. Drag the install button  to Eclipse workspace

 

2. We should see now the Eclipse Marketplace window. We are going to install all the features, so leave all the default values and click Confirm.

 

3. When get prompted to restart Eclipse, select Yes

 

 

Bluemix Server Configuration

1. On the Serves tab, right click on any empty space, and select New > Server

 

2. Expand IBM,then select IBM Bluemix. Click Next

 

3. Enter Bluemix account login Email and Password. Click Finish

 

4. Select your Organization and Space. Click Finish

 

5. You should now see IBM Bluemix server appearing started in the Servers tab

 

 

Clone and Import Project from GitHub

1. In Eclipse workspace, click File > Import. Expand Git, and select Projects from Git. Click Next

 

2. Select Clone URI, then click Next

 

3. In the URI column, enter https://hub.jazz.net/git/ecosysdevcnc/javaplays-restjava-basic, then click Next.

 

4. Make sure master branch is checked, then click Next

 

5. Accept the default values, and click Next

 

6. Leave all the default values, and click Next

 

7. Leave all the default values again, and click Finish.

 

8. After the project successfully imported, go to Markers tab, you should see bunch of errors, in which we going to resolve in next section.

 

 

Fix Project Buildpath

1. In Eclipse Worksapce, at Project Explorer view, right click on Project javaplays-restjava-mobilestarterkit-templatecode, and select Build Path > Configure Build Path

 

2. Make sure you are at Libraries tab. You should see there is an error linking to unbound JRE library. Select the JRE System Library [Java SE 7 [1.7.0_75]] (unbound), and click Edit button

 

3. Select Workspace default JRE (jdk 1.7.0_79), and click Finish

 

4. The error should get rectified. Click OK

Deploy Project to WAS Liberty Server (local)

1. In Eclipse Workspace, at Enterprise Explorer view, right click on Project javaplays-restjava-mobilestarterkit-templatecode, and select Run As > Run on Server

 

2. Make sure you select Choose an existing server, Expand localhost, select WebSphere Application Server Liberty Profile at localhost, and click Finish

 

3. Embedded browser of Eclipse appears and display simple web page with message “Hello!!”

Test Local Application with Postman

1. Go to Postman Chrome extension page. Change to POST, and enter url http://localhost:9080/javaplays-serverside-mobilestarterkit-template/api/service/login

Click on Body tab, select raw, and enter the following:

{

        “user_id”: “admin”,

        “password”: “password”

}

Finally press the Send button

 

2. You should see the message Successful at the bottom of the page

Deploy Project to Remote WAS Liberty Server (Bluemix)

1. In Eclipse Workspace, at Enterprise Explorer view, right click on Project javaplays-restjava-mobilestarterkit-templatecode, and select Run As > Run on Server

 

2. Make sure Choose an existing server is checked. Expand Cloud, and select IBM Bluemix. Click Finish

 

3. For Name field, we need to provide a global unique name, then click Finish.
    If the name is taken by someone else (or perhaps by ourselves), error message will pop out and asking to provide another host name.

 

4. If the application is successfully deployed to Bluemix server, you should see the simple web page with message “Hello!!!”

Test Remote Bluemix Application with Postman

1. Go to Postman Chrome Extension page. Make sure POST is selected, and enter url http://<global-unique-name>.mybluemix.net/api/service/login (Global-unique-name refer to the name defined at Step 3 during application deployment to Bluemix server)

Click on Body tab, select raw, and enter the following:

{

        “user_id”: “admin”,

        “password”: “password”

}

Finally press the Send button. You should see the message “Successful” at the bottom of the page.

 

 

2. Change the url to http://<global-unique-name>.mybluemix.net/api/service/searchtips. In the Body tab, change the input to the following:
